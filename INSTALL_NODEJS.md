# Установка Node.js для EkiTili

Node.js необходим для работы бэкенд сервера, который подключается к Oracle Database.

## Что такое Node.js?

Node.js - это среда выполнения JavaScript, которая позволяет запускать JavaScript код на сервере. В нашем проекте она нужна для:
- Запуска бэкенд API сервера (подключение к Oracle)
- Установки зависимостей проекта (npm пакеты)

## Шаг 1: Скачивание Node.js

1. Откройте браузер и перейдите на официальный сайт:
   **https://nodejs.org/**

2. На главной странице вы увидите две версии:
   - **LTS (Long Term Support)** - рекомендуемая версия (стабильная)
   - **Current** - последняя версия (может содержать новые функции)

3. **Рекомендуется скачать LTS версию** - нажмите на большую зеленую кнопку "LTS"

4. Будет скачан установочный файл (например, `node-v20.x.x-x64.msi`)

## Шаг 2: Установка Node.js

1. **Запустите скачанный файл** (обычно находится в папке "Загрузки")

2. Нажмите **"Next"** на экране приветствия

3. **Примите лицензионное соглашение** и нажмите "Next"

4. **Выберите путь установки** (можно оставить по умолчанию: `C:\Program Files\nodejs\`) и нажмите "Next"

5. **ВАЖНО: Настройка компонентов**
   - Убедитесь, что отмечены все пункты:
     - ✅ Node.js runtime
     - ✅ npm package manager
     - ✅ Add to PATH (это важно!)
   - Нажмите "Next"

6. Нажмите **"Install"** (может потребоваться разрешение администратора)

7. Дождитесь завершения установки и нажмите **"Finish"**

## Шаг 3: Проверка установки

1. **Закройте все окна командной строки/терминала** (если они были открыты)

2. Откройте **новое** окно командной строки:
   - Нажмите `Win + R`
   - Введите `cmd` и нажмите Enter

3. Проверьте версию Node.js:
   ```bash
   node --version
   ```
   Должно показать что-то вроде: `v20.x.x`

4. Проверьте версию npm (менеджер пакетов):
   ```bash
   npm --version
   ```
   Должно показать что-то вроде: `10.x.x`

Если обе команды работают - Node.js установлен правильно! ✅

## Шаг 4: Установка зависимостей проекта

После установки Node.js нужно установить зависимости для бэкенд сервера:

1. Откройте командную строку в папке проекта:
   ```bash
   cd D:\_work\ekitili-main
   ```

2. Установите зависимости:
   ```bash
   npm install
   ```

3. Дождитесь завершения установки. Вы увидите сообщение о создании папки `node_modules`

## Что дальше?

После установки Node.js и зависимостей:

1. **Настройте подключение к Oracle** в файле `server.js` (укажите ваш пароль)
2. **Установите Oracle Instant Client** (см. [SETUP_ORACLE.md](SETUP_ORACLE.md))
3. **Запустите серверы** через `start-server.bat`

## Решение проблем

### Проблема: "node не является внутренней или внешней командой"

**Причина:** Node.js не добавлен в PATH или терминал был открыт до установки.

**Решение:**
1. Перезагрузите компьютер (или хотя бы закройте и откройте терминал заново)
2. Проверьте, что Node.js установлен: откройте "Программы и компоненты" и найдите Node.js
3. Если не помогло, переустановите Node.js и убедитесь, что отмечен пункт "Add to PATH"

### Проблема: "npm не найден"

**Причина:** npm не установился вместе с Node.js.

**Решение:**
1. Переустановите Node.js
2. Убедитесь, что при установке отмечен пункт "npm package manager"

### Проблема: Ошибки при `npm install`

**Возможные причины:**
- Нет интернета
- Проблемы с правами доступа
- Антивирус блокирует установку

**Решение:**
1. Запустите командную строку от имени администратора
2. Попробуйте снова: `npm install`
3. Если не помогает, попробуйте: `npm install --force`

### Проблема: Установка очень медленная

**Решение:**
Это нормально при первой установке. npm скачивает много пакетов. Подождите, процесс может занять несколько минут.

## Альтернативный способ: Установка через Chocolatey

Если у вас установлен Chocolatey (менеджер пакетов для Windows):

```bash
choco install nodejs-lts
```

## Альтернативный способ: Установка через winget

Если у вас Windows 10/11 с winget:

```bash
winget install OpenJS.NodeJS.LTS
```

## Проверка после установки

После установки выполните в командной строке:

```bash
node --version
npm --version
```

Обе команды должны показать версии без ошибок.
